Prompt_Acception
### Contexte

Tu disposes d'un corpus de conversations. Pour chaque conversation, toutes les informations nécessaires sont incluses directement dans l'échange :

- **Le locuteur** qui utilise l'expression cible
- **Les interlocuteurs**
- **La conversation complète** (tours de parole)
- **L'expression PPI** à analyser apparaît dans son contexte naturel

### Mission

Pour chaque conversation du corpus, tu dois identifier **l'acception** (le sens spécifique) de l'expression PPI ciblée dans son contexte conversationnel, en te basant sur le référentiel d'acceptions défini ci-dessous.

### Référentiel d'acceptions (exemples types)

- **ça va** peut avoir l'acception : `salutation` ou `exaspération`
- **c’est pas vrai** peut avoir l'acception : `surprise` ou `faux/fausseté`
- Adapte ce référentiel mentalement à l'expression analysée dans le corpus

### Format de sortie attendu

Pour chaque conversation analysée, produis une entrée structurée avec :

- **Propriété** : [glose courte, max 2 mots, descriptive]
- **Justification** : [brève justification contextuelle, 1 phrase max]

**Exemple de sortie :**
```json
[
  {
    "Propriété": "salutation",
    "Justification": "Ouverture de l'échange, sans référence à un état antérieur"
  },
  {
    "Propriété": "exaspération",
    "Justification": "Réponse à une longue plainte de l'interlocuteur, marque la lassitude"
  }
]
```

### Critères d'analyse (pour chaque conversation)

1. **Identifie l'acception pertinente** :
   - Consulte **uniquement le contexte fourni** (le tour de parole contenant l'expression, et les tours adjacents si nécessaire).
   - Si l'expression, selon le référentiel, peut avoir plusieurs acceptions très divergentes, **détermine celle qui est activée par le contexte spécifique de cette conversation**.
   - Si l'expression **n'est pas listée comme polysémique** dans ton référentiel mental ou si son sens est **unique et évident**, indique : `Sens unique`.

2. **Formule l'acception par une glose courte (max 2 mots) et descriptive** :
   - Utilise une glose claire et précise, sur le modèle des exemples (`surprise`, `fausseté`, `salutation`, `exaspération`).
   - La glose doit **décrire le sens porté par l'expression dans cet échange spécifique**.

3. **Méthode d'analyse par conversation** :
   - Concentre-toi sur **l'intention du locuteur** et **l'effet dans la conversation**.
   - Observe **ce qui précède et ce qui suit immédiatement** l'expression : à quoi répond-elle ? Quel acte de parole accomplit-elle ?
   - **Ne confonds pas** l'acception avec la **modalité grammaticale** (interrogative, exclamative...).

4. **Traitement de l'ensemble du corpus** :
   - Analyse systématiquement chaque conversation.
   - Identifie les patterns : une même expression peut avoir des acceptions différentes selon les contextes.
   - Sois cohérent dans tes choix d'acception pour des contextes similaires.

### Note importante

- Le paramètre **acception** distingue des **sens radicalement différents** pour une même expression, pas des nuances fines.
- Limite-toi à l'analyse de la modalité et du sens pragmatique de l'expression, pas aux subtilités discursives avancées.
- Une même expression peut avoir l'acception `Sens unique` dans tout le corpus si elle n'est pas polysémique.

---
end_prompt

Prompt_TypePhrase
### Contexte

Tu disposes d'un corpus de conversations. Pour chaque conversation, toutes les informations nécessaires sont incluses directement dans l'échange :

- **Le locuteur** qui utilise l'expression cible
- **Les interlocuteurs**
- **La conversation complète** (tours de parole)
- **L'expression PPI** à analyser apparaît dans son contexte naturel

### Mission
* ta tâche est de déterminer si l'expression cible est autonome (clausative) ou facultative (parenthétique)
les parenthétiques fonctionnent généralement comme des commentaires insérés dans le discours et qui peut être effacé, e.g: 
	la vie tu vois est dure --> tu vois (parenthétique)
	la vie, comment dirais-je, est dure --> comment dirai-je est parenthétique
	le foot, c'est pas mon truc --> "c'est pas mon truc" causative car ce n'est pas seulement un commentaire facultatif.
Pour chaque conversation du corpus, tu dois déterminer si l'expression PPI est de type **Clausative** ou **Parenthétique**, en te basant sur son fonctionnement syntaxique dans le tour de parole où elle apparaît.

### Définitions

- **Clausative (phrase autonome)** :
  - L’expression constitue elle-même une phrase typographiquement complète : elle débute la phrase, éventuellement suite à des marqueurs de discours, des appellatifs ou des adverbes et se termine par un point, un point-virgule, un point d’exclamation, un point d’interrogation ou des points de suspension. Exemple : *Je dois partir. Au revoir.*
  - Segment syntaxique **autonome**, avec une tête prédicative complète.
  - Possède sa propre **modalité d’énonciation**.
  - Peut constituer une **proposition principale** pour une subordonnée. Exemple : *<PPI> comment se fait-il</PPI> que tu sois là*.
  - Exemples : *Ah au secours !*, *C’est clair.*, *Mais, tu plaisantes ?*, *Franchement, il manquerait plus que ça…*
  - **Test** : Peut être énoncée seule sans autre élément.

- **Parenthétique (prédication seconde)** :
  - Insérée dans une autre phrase sans lien de subordination et entourée dans les textes écrits de virgules ou virgule et point final. Exemple : *Je suis fatigué, <PPI> c'est le mot</PPI>.*
  - **Insérée** dans une autre phrase, sans lien de subordination.
  - **Facultative** : son effacement ne rend pas la phrase agrammaticale.
  - Fonctionne souvent comme un **commentaire méta-énonciatif**.
  - Exemples : *Il était, tu vois, mort de rire.*, *C’est un problème, c’est clair.*, *C’est un, comment dirais-je, ennui.*
  - **Test** : Peut être supprimée sans altérer la structure de base.

### Format de sortie attendu

Pour chaque conversation analysée, produis une entrée structurée avec :

- **Type** : "Clausative" ou "Parenthétique"
- **Justification** : [brève justification, 1 phrase, basée sur les tests syntaxiques]

**Exemple :**
```json
[
  {
    "Propriété": "Clausative",
    "Justification": "L'expression constitue une phrase autonome, elle peut être énoncée seule."
  },
  {
    "Propriété": "Parenthétique",
    "Justification": "L'expression est insérée dans une autre phrase et peut être supprimée sans perte de grammaticalité."
  }
]
```

### Critères d'analyse (pour chaque conversation)

1. **Isole l'occurrence** de l'expression dans le tour de parole du locuteur.
2. **Applique le test de suppression** : si l'expression peut être retirée sans rendre la phrase agrammaticale et qu'elle est syntaxiquement intégrée à une autre proposition, c'est **Parenthétique**. Sinon, si elle forme une phrase autonome, c'est **Clausative**.
3. **Justifie** ton choix en une phrase.

### Note importante

- Ne tiens pas compte du sens ou de l'intonation, uniquement de la structure syntaxique.
- Si l'expression est une proposition principale avec une subordonnée (ex: "comment se fait-il que..."), elle est **Clausative**.



### exemples d'erreurs fréquentes à éviter

* Entrée: Tour de parole [Locuteur 1] Bon, ça suffit. Les funérailles, <PPI>c est pas mon truc</PPI>,
* Sortie: Parenthétique

* Problème: la PPI  <PPI>c'est pas mon truc</PPI> même si elle est entourée de virgules, elle n'est pas parenthétique car elle n'est pas optionnelle dans la phrase: on ne peut pas dire seulement Les funérailles. (la phrase est incomplète syntaxiquement
* solution: considérer cette PPI comme "Clausative"

Prompt_Modalite
### Contexte

Tu disposes d'un corpus de conversations. Pour chaque conversation, toutes les informations nécessaires sont incluses directement dans l'échange :

- **Le locuteur** qui utilise l'expression cible
- **Les interlocuteurs**
- **La conversation complète** (tours de parole)
- **L'expression PPI** à analyser apparaît dans son contexte naturel

### Mission

Pour chaque conversation du corpus, identifie la **modalité d’énonciation** de l'expression PPI, basée sur sa **structure syntaxique/morphologique**.

### Catégories possibles

- **Interrogative** :
  - Structure interrogative (inversion, mot interrogatif, point d’interrogation à l’écrit).
  - Exemples : *Comment oses-tu ?*, *Ça va ?*

- **Déclarative** :
  - Structure affirmative ou négative, souvent accompagnée d’un point ou d’un point d’exclamation à l’écrit.
  - **Inclut** les énoncés perçus comme exclamatifs (ex: *C’est le pied !*).
  - Exemples : *C’est pas vrai.*, *À bientôt.*

- **Injonctive** :
  - Mode impératif ou subjonctif d’ordre/exhortation.
  - Exemples : *Épargne-moi les détails.*, *À table !*

### Format de sortie attendu

Pour chaque conversation analysée, produis une entrée structurée avec :

- **Modalite** : "Interrogative", "Déclarative" ou "Injonctive"
- **Justification** : [brève justification basée sur la forme]

**Exemple :**
```json
[
  {
    "Propriété": "Interrogative",
    "Justification": "Présence d'un mot interrogatif 'comment' et d'une inversion sujet-verbe."
  }
]
```

### Critères d'analyse

- Examine la **structure syntaxique** de l'expression (présence d'un mot interrogatif, mode du verbe, ponctuation).
- Attribue la modalité correspondante.
- Justifie en mentionnant l'indice formel.

### Note importante

- La modalité **exclamative** n'est pas une catégorie distincte ; les énoncés exclamatifs sont classés en **Déclarative**.
- Pour les corpus écrits, tiens compte de la **ponctuation** (?, !).

---
end_prompt

Prompt_ParticularitesSyntaxiques
### Contexte

Tu disposes d'un corpus de conversations. Pour chaque conversation, toutes les informations nécessaires sont incluses directement dans l'échange :

- **Le locuteur** qui utilise l'expression cible
- **Les interlocuteurs**
- **La conversation complète** (tours de parole)
- **L'expression PPI** à analyser apparaît dans son contexte naturel
- **La forme canonique (lemme)** de l'expression est également fournie
- **La forme occurrence** (la variante réellement employée) est celle qui apparaît dans la conversation

### Mission

Pour chaque conversation, compare la **forme occurrence** à la **forme lemme** de l'expression et identifie les différences syntaxiques éventuelles.

### Consignes d'analyse

- Compare la variante **(forme occurrence)** à sa forme par défaut **(lemme)** et détermine s'il y a des différences observées dans la forme, la structure syntaxique, etc.
  - **Deux cas :**
    a. Il n'y a **pas de différences**.
    b. Il y a **des différences** : cite-les.

- **Note les différences** de pronoms (ex: *tu* --> *vous*), les inversions du sujet, les ajouts/suppressions de mots, les variations de temps/mode, etc.
- **Limite-toi aux aspects syntaxiques** : n'aborde pas le sens, l'usage, le registre, les formes orales/écrites. Il faut indiquer le type de structure dans la réponse.

#### Cas particuliers à traiter :

- **Interrogatives partielles (avec mot interrogatif)** : Indiquer la structure selon le tableau :

| Structure | Exemple |
|-----------|---------|
| in situ | Ils sont partis où ? |
| Antéposition | Où ils sont partis ? |
| Inversion du clitique | Où sont-ils partis ? |
| Inversion complexe | Où les autres sont-ils partis ? |
| Inversion stylistique | Où sont partis les autres ? |
| Clivage | C'est où qu'ils sont partis ? |
| 'Est-ce que' | Où est-ce qu'ils sont partis ? |
| Variante de 'est-ce que' | Où c'est qu'ils sont partis ? |
| Complémenteur | Où qu'ils sont partis ? |
| Sujet qu- | Lesquels sont partis ? |

- **Interrogatives totales** (réponses en oui ou non) : Noter la structure.

| Structure | Exemple |
|-----------|---------|
| Intonation | Ils/Les autres sont partis ? |
| Est-ce que | Est-ce que les autres/ils sont partis ? |
| Inversion du clitique | Sont-ils partis ? |
| Inversion complexe | Les autres sont-ils partis ? |

- **Dislocation à gauche et à droite** : La dislocation apparaît quand on reprend un élément de la phrase avec un syntagme. À l’écrit, l’élément disloqué est généralement séparé par une virgule.

| Structure | Exemple |
|-----------|---------|
| Dislocation gauche | **ce truc**, *c'est* un problème. |
| Dislocation droite | *c'est* un problème **ce truc** |

- **Phrase dans une subordonnée** : La PPI apparaît dans une subordonnée complétive en *que* ou *si*.

| Structure | Exemple |
|-----------|---------|
| Complétive | Je trouve que *c'est* un problème. |

### Format de sortie attendu

Pour chaque conversation analysée, produis une entrée structurée avec :

- **Differences** : description des différences syntaxiques, ou "Aucune différence"

**Exemple :**
```json
[
  {
    "Propriété": "Ajout de 'non ?' en incise, ce qui ajoute une modalité interrogative.",
    "Justification": "La forme détectée présente un ajout de la particule non? qui introduit une modalité interrogative différente de celle relevée dans le lemme"
  },
  {
    "Propriété": "Changement de pronom : tu -> vous (forme de politesse ou pluriel).",
    "Justification": "Le pronom tu du lemme a été substitué par vous"
  }
]
```

### Note importante

- Ne fais **aucune analyse sémantique ou pragmatique**.
- Si la forme est identique au lemme, indique **"Aucune différence"**.

Prompt_Expansions
### Contexte

Tu disposes d'un corpus de conversations. Pour chaque conversation, toutes les informations nécessaires sont incluses directement dans l'échange :

- **Le locuteur** qui utilise l'expression cible
- **Les interlocuteurs**
- **La conversation complète** (tours de parole)
- **L'expression PPI** à analyser apparaît dans son contexte naturel


* ta tâche: repérer les expansons de l'expression cible:

* l'expansion est un élément qui vient toujours après et c'est un complément régi syntaxiquement par l'expression cible, e.g: comment ça se fait que tu sois ici--> que tu sois ici est une expansion
* Un élément linguistique n'est considéré comme expansion que si et seulement si il fait partie de ces trois types:

    • De structures infinitives : comment oses-tu dire cela ?
    • De phrases en que : c’est clair que Léa vient demain.
    • De complément nominaux ou prépositionnels : tu parles d’un spectacle.
si tu trouve tout élément en dehors de ces trois types, il faut indiquer "Aucune expansion" 

Exemples d'éléments qui ne sont pas des expansions:
* les éléments disloqués: les fleurs, je les aime beaucoup --> 'je les aime beaucoup' n'est pas une expansion, c'est un élément disloqué
 
 
* le lien syntaxique doit être évident
* dans les dialogues authentiques, il n y a pas de ponctuation, il faut inférer le lien syntaxique du contexte 
* si l'élément immédiatement après l'expression cible est juste juxtaposé sans lien syntaxique, indiquer 'Aucune expansion'
* à ne pas confondre avec les éléments disloqués, e.g: c'est beau la vie--> la vie = élément disloqué: il n'est pas régi synatiquement par c'est beau

    
* format de réponse attendu:
Pour chaque conversation analysée, produis une entrée structurée avec :
{
  "Propriété": "[élément linguistique trouvé] ou [Aucune expansion]",
  "Justification": "justification de la réponse, et tour de parole complet avec l'expansion entourée de balises <EXP> e.g: tour de parole complet: <PPI>comment peux-tu</PPI><EXP>faire ceci</EXP>"
}

**Exemple :**
```json
[
  {
    "Propriété": "Proposition en 'que' : <EXP>'que Léa vient demain'</EXP>",
    "Justification": "L'expression est suivie d'une subordonnée complétive introduite par 'que'."
  },
  {
    "Propriété": "Aucune expansion",
    "Justification": "L'expression est employée seule, sans complément régi."
  }
]
```

### Critères d'analyse

- Examine le **contexte immédiat** de l'expression dans le tour de parole.
- Vérifie si elle est suivie (ou précédée) d'un complément **régi syntaxiquement** (c'est-à-dire obligatoire pour compléter le sens, pas un simple modifieur).
- Si oui, **décris-le** (type et contenu). Sinon, indique **"Aucune expansion"**.
- **Limite-toi aux aspects syntaxiques**.

### Note importante

- Évite d'utiliser le formatage JSON à l'intérieur des propriétés.

---
end_prompt

Prompt_Modifieurs
### Contexte

Tu disposes d'un corpus de conversations. Pour chaque conversation, toutes les informations nécessaires sont incluses directement dans l'échange :

- **Le locuteur** qui utilise l'expression cible
- **Les interlocuteurs**
- **La conversation complète** (tours de parole)
- **L'expression PPI** à analyser apparaît dans son contexte naturel


* ta tâche: repérer les modifieurs de l'expression cible (e.g très, trop, extrêmement, baucoup --> c'est pas grave --> c'est pas très grave)

ATTENTION: - le modifieur est un élément supplémentaire qui s'ajoute à l'expression, il ne fait pas partie d'elle d'origine: par exemple: le mot 'pas' dans 'c'est pas grave' N'EST PAS UN MODIFIEUR!, car il fait partie intégrante de l'expression elle meme,
 	   -'très' dans 'c'est pas très grave' est un modifieur car c'est un élément ajouter pour intensifier l'expression


un modifieur est un élément qui modifie l'expression, il peut être inséré ou bien à la fin de l'expression, comme les adverbes.
* on considère un élément comme un modifieur si et seulement si il est ajouté à l'expression d'origine et il l'intensifie ou la modalise, e.g: c'est très bien--> 'très' est un modifieur 
* le modifieur est soit inséré dans l'expression, soit immédiatement postposé, il n'est jamais séparé de l'expression cible.

{
  "Propriété": "[élément linguistique trouvé] ou [Aucun modifieur]",
  "Justification": "justification de la réponse, et tour de parole complet avec le modifieur entourée de balises <MOD> e.g: tour de parole complet: <PPI>comment peux-tu</PPI><EXP>faire ceci</EXP>"
}

**Exemple :**
```json
[
  {
    "Propriété": ["bien (postposé)"],
    "Justification": "L'expression est suivie de l'adverbe 'bien' qui intensifie le sens."
  },
  {
    "Propriété": "Aucun modifieur",
    "Justification": "L'expression apparaît sans élément modificateur."
  }
]
```

### Critères d'analyse

- Examine le **contexte immédiat** (gauche/droite) de l'expression dans le tour de parole.
- **Liste** les modifieurs présents (adverbes, locutions).
- **Indique leur position** (antéposé = avant, postposé = après).
- Si aucun, indique **"Aucun modifieur"**.

Prompt_Coocurrents
### Contexte

Tu disposes d'un corpus de conversations. Pour chaque conversation, toutes les informations nécessaires sont incluses directement dans l'échange :

- **Le locuteur** qui utilise l'expression cible
- **Les interlocuteurs**
- **La conversation complète** (tours de parole)
- **L'expression PPI** à analyser apparaît dans son contexte naturel

### role général:
* ta tâche c'est de trouver des éléments qu'on appelle des cooccurrents qui fonctionnent avec l'expression à analyser

* par exemple: bon ben, à bientôt, donc euh

--> le marqueur du discours 'bon ben' est un cooccurrent de l'expression 'à bientôt'
--> le marqueur du discours 'donc eu' est un cooccurrent de l'expression 'à bientôt'

* un élément linguistique est considéré comme un cooccurrent si et seulement si il a un lien direct avec l'expression cible, e.g: eh bien, c'est pas grave--> eh bien est étroitement lié à 'c'est pas grave'
* Exemple de réponse erronnée: entrée: "c'est pas grave eh bien comment ça va" --> sortie: "marqueur de discours:eh bien' --> la réponse est erronnée car "eh bien" n'est pas lié à "c'est pas grave"  mais il est lié à l'expression qui la suit 'comment ça va' --> donc il n'est pas un cooccurrent de 'c'est pas grave'.
* le sens du cooccurrent doit vraiment porter sur l'expression cible.
* Il faut indiquer le cooccurrent si et seulement si il porte sémantiquement et pragmatiquement directement sur l'expression cible comme dans les deux exemples indiqués, autrement, indiquer "Aucun cooccurrent"
* par fois il y a des combinaisons de marqueurs de discours, e.g: eh ben, bah alors, etc.
* il ne faut retenir que les cooccurrents les plus proches de l'expression cible et qui ne sont pas séparés syntaxiquement de celle-ci
* à l'oral spontanné, il ne faut pas retenir les cooccurrents qui font partie d'une phrase tronquée et incomplète qui précède ou suit l'expression cible.
* parfois les dialogues sont authentiques et contiennent des hésitations, donc prendre ça en considération, e.g: "mais c'est pas c'est pas grave"--> "mais" est cooccurrent de 'c'est pas grave" car "c'est pas" est juste une hésitation de l'expression cible.

IMPORTANT: tout élément qui n'est pas immédiatement antéposé ou postposé à l'expression cible N'EST PAS UN COOCCURRENT!

* workflow à suivre:

	1. identifier l'expression cible
	2. chercher 3 mots immédiatement avant et 3 mots immédiatement après
	3 pour chacun de ces 3 mots : 
		s'il introduit l'expression cible --> c'est un cooccurrent
		s'il est juste proche parce que c'est un dialogue oral et il n y a pas de ponctuation pour le séparer de l'expression --> c'est pas un cooccurren
		s'il introduit le mot qui le suit et non pas l'expression cible --> c'est pas un cooccurrent (e.g bon ben c'est bien c'est pas grave)
		s'il fonctionne avec le/les mots qui le suivent comme un seul bloc et le bloc introduit l'expression cible --> c'est des cooccurrents (e.g bon ben c'est pas grave)
		si l'appellatif fonctionne avec un autre segment que l'expression cible --> c'est pas un cooccurrent (e.g ben écoute Jean c'est pas grave)
je t'indique quelques exemples de cooccurrents posibles, mais tu dois trouver aussi ceux qui ne figurent pas dans cette liste:

| Catégorie | Exemples |
|-----------|----------|
| **Appellatifs** | Monsieur, Jean, Jeanne, Chéri, Maman, etc. |
| **Adjectifs** | bon, chic, chiche, tout beau, tout doux |
| **Adverbes** | alors, assez, ben, bien, comment, donc, effectivement, enfin, hélas, jamais de la vie, là, quand même, tant mieux, tant pis |

| **Noms** | adieu, attention, banco, bingo, bonjour, bonsoir, bonté, bordel, bougre, camembert, catastrophe, ciel, chapeau, chiotte, Christ, con, courage, crotte, dame, diable, dommage, genre, grâce, flûte, halte, malheur, merci, merde, misère, nom de Dieu, Jésus, pardon, pétard, pitié, prout, punaise, purée, putain, respect, salut, santé, seigneur, stop, tintin, tralala, tonnerre |

Prompt_Declenchement
### Contexte

Tu disposes d'un corpus de conversations. Pour chaque conversation, toutes les informations nécessaires sont incluses directement dans l'échange :

- **Le locuteur** qui utilise l'expression cible
- **Les interlocuteurs**
- **La conversation complète** (tours de parole)
- **L'expression PPI** à analyser apparaît dans son contexte naturel

### Mission
ta tâche: déterminer l'élément linguistique ou extralinguistique qui a provoqué la production de l'expression cible
Pour chaque conversation du corpus, identifie la **catégorie de déclenchement** de l'expression PPI

### Catégories possibles

1. **HETERO (Hétérodéclenché)** :
   - La PPI est une **réaction directe** à un **stimulus verbal** de l’interlocuteur dans le tour précédent.
   - Exemple : *– Merci. – Y’a pas de quoi.*

2. **NEG (Non hétérodéclenché)** :
   - Cette catégorie regroupe deux cas :
     - **Autodéclenchement** : le stimulus est **à l’intérieur du même tour** du locuteur.
       Exemple : *Franchement, c’est la cata.*
     - **Non déclenché** : la PPI **initie un échange** (salutation, changement de sujet).
       Exemple : *Salut !*, *Au fait, tu sais quoi ?*

3. **CITATION (Déclenché par hétérorépétition)** :
   - Le locuteur **reprend/rapporte** les propos de l’interlocuteur dans son propre tour, puis y réagit. Il faut que les propos répétés soient synonymes ou très proches.
   - Fréquent dans les débats, forums, dialogues romanesques.
   - Exemple : `[locuteur 1] Elle a eu 6 enfants [locuteur 2] Six enfants ? <PPI>Vous plaisantez</PPI>`

4. **EXTRA (Déclenché par la situation extralinguistique)** :
   - La PPI est provoquée par un **élément contextuel non verbal** (perception, situation).
   - Exemples : En voyant quelqu’un : *Quelle bonne surprise !* ; Après une description dans un roman : `<PPI>C’est pas croyable !</PPI>`

### Format de sortie attendu

Pour chaque conversation analysée, produis une entrée structurée avec :

- **Declenchement** : "HETERO", "NEG", "CITATION", ou "EXTRA"
- **Justification** : [brève, précisant le stimulus et sa source]

**Exemple :**
```json
[
  {
    "Propriété": "HETERO",
    "Justification": "Réaction directe au remerciement de l'interlocuteur dans le tour précédent."
  },
  {
    "Propriété": "NEG",
    "Justification": "L'expression initie un nouvel échange (changement de sujet), sans stimulus antérieur."
  }
]
```

### Critères d'analyse

- Analyse le **contexte conversationnel** (tours précédents).
- Si l'expression répond à un énoncé de **l'interlocuteur immédiatement avant** → **HETERO**.
- Si elle **reprend les mots de l'interlocuteur** (citation) → **CITATION**.
- Si elle est déclenchée par une **situation physique** (ex: voir quelqu'un) → **EXTRA**.
- Sinon (autodéclenchée dans le même tour ou initiative) → **NEG**.
- **Limite-toi au déclenchement**, pas d'analyse sémantique.
- Si cela paraît difficile à décider, indique **NSP (ne sais pas)**.
- **Ton analyse doit être basée sur le contenu de la conversation fournie.**

### Typologie à utiliser

1. **PR (Pragmatèmes / Phrases rituelles)** :
   - **PRS** : Salutations (ex: *bonjour*, *ça va ?*, *à la prochaine*)
   - **PRP** : Formules de politesse (ex: *merci*, *je vous en prie*, *bon voyage*)
   - **PRA** : Autres phrases rituelles spécifiques (ex: *qui est à l’appareil ?*, *et avec ceci ?*)

```json
  }
]
```
